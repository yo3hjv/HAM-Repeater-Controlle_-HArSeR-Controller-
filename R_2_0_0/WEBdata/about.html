About ESP32 Repeater Controller

About the Project

The ESP32 Repeater Controller is a project designed to provide a flexible, feature-rich controller for amateur radio repeaters. Built on the ESP32 platform, it offers WiFi connectivity, web-based configuration, and comprehensive repeater control features.

Key features include:

  • RSSI-based carrier detection with adjustable hysteresis
  • Can be locked from Web interface
  • Configurable hang time and timeout settings
  • Automatic complex CW beacon with customizable text and timing
  • Courtesy and tail tone generation
  • WiFi connectivity with both AP and STA modes
  • Web-based configuration interface
  • Optional LCD display support

Settings Explanation

RSSI Hysteresis

In RSSI mode, the RSSI (Received Signal Strength Indicator) settings control how the repeater detects incoming signals.
The threshold determines the minimum signal level required to trigger the repeater, while the hysteresis ensure a stable operation while signal is fluctuating in intensity.

Timing Settings

A second Hyseteresis, time-based, maintain a stable operations and prevent kerchunking.
Hang Time: The period the repeater remains active after the input signal disappears.
Timeout: Maximum transmission time allowed before the repeater temporarily disables.
Timeout Penalty: The duration the repeater remains disabled after a timeout occurs.

Beacon Settings

The beacon automatically transmits your callsign or other identification at regular intervals. You can customize the message text, transmission interval, CW speed, and tone frequency. Setting the interval to 0 disables automatic beacons.

Beacon Format is: [Callsign] + [Delay] + [End Message]
The delay between callsign and end message can be adjusted, and different end messages can be configured for when the repeater is active or locked.

Courtesy Tone

The courtesy tone plays briefly when a transmission ends and the hang time begins. It signals to users that the repeater is ready for another transmission.

Tail Tone

The tail tone plays at the end of the hang time, just before the repeater deactivates. It signals that the repeater is about to go into standby mode.

Preferences & Settings

Settings Persistence

All settings are automatically saved to the Repeater controller's non-volatile memory and persist across power cycles and reboots. You can download a complete backup of your settings using the green "Download Settings" button on the Settings page. This creates an .ini file that can be used to restore your configuration.

For troubleshooting, you can enable detailed debug messages for preferences saving and loading operations on the Debug page. This helps verify that your settings are being properly stored and retrieved.

A complete list of all persistent settings and their descriptions can be found in the Persistent_Settings.md file in the project repository.

Hardware Information

Connections

ESP32 Pin Function Description
GPIO 36 RSSI Input Analog input for receiver signal strength
GPIO 14 Carrier Detect Digital input for carrier detection
GPIO 12 PTT Output Controls transmitter PTT
GPIO 25/26/27 Audio Output For tones and beacon (user selectable)
GPIO 23 Beacon Trigger External beacon trigger input
GPIO 2 TX LED Transmit status indicator
GPIO 21 I2C SDA For LCD display
GPIO 22 I2C SCL For LCD display

Hardware Requirements

  • ESP32 development board
  • Optional: I2C LCD display (16x2 or 20x4)
  • Interface circuit for connecting to radio equipment
  • Stable power supply
